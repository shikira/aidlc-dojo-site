# AI-DLC DOJO サイト

AI-DLC DOJO 静的学習サイトの**コードとコンテンツ**を管理する公開リポジトリです。

> このリポジトリはサイトのコードと教材コンテンツのみを含みます。AI-DLC の内部作業記録(監査ログ・診断・`aidlc/` ワークスペース等)は含みません(ADR-2 / 公開境界ルール P1。`.gitignore` で構造的にも除外)。

## 現在の状態(ブートストラップ)

このリポジトリは現在 **UW-06a(リポジトリ・ブートストラップ)** の成果物、すなわち **CI 骨格とリポジトリ設定の「器」** のみを含みます。Astro アプリ本体(ページ・コンポーネント)は **UW-01 以降**が追加します。

そのため `build` は**プレースホルダ**(`bootstrap placeholder — replaced by UW-01` を出力して終了)です。UW-01 が同名の `build` コマンドを実装した時点で、コマンドセット契約を通じて自動的に本物のビルドへ置き換わります。

## コマンドセット契約

CI はプロジェクト標準のコマンド名を起動する契約で構成されています。後続 Unit はこれらのコマンドの実体を差し込むだけで CI ゲートに乗ります。

| コマンド        | 内容                                     | 現時点の実体                                             |
| --------------- | ---------------------------------------- | -------------------------------------------------------- |
| `npm run lint`  | Prettier(整形チェック)+ ESLint(静的検査) | 空プロジェクトでも成立                                   |
| `npm test`      | Vitest 実行 + v8 カバレッジ計測          | テスト0件でも緑(80% 閾値は `src/**/*.ts` 存在時のみ活性) |
| `npm run build` | 本番ビルド                               | プレースホルダ(UW-01 まで)                               |

補助コマンド:

- `npm run format` — Prettier で自動整形
- `npm run test:watch` — Vitest をウォッチモードで実行

### 前提

- Node.js **20 以上**

### セットアップ

```sh
npm ci        # ロックファイルに基づく再現インストール
npm run lint
npm test
npm run build
```

## マージ規律

- `main` への直接コミットは禁止(ブートストラップ窓を除く)。変更は Pull Request 経由。
- マージは **squash のみ**(1 変更 = `main` 上の 1 コミット)。
- 唯一の必須ステータスチェックは集約ジョブ **`ci-gate`**(`lint` / `test` / `build` の全成功に依存)。
- CI 失敗(lint / test / build のいずれか)ではマージできません。

ブランチ保護・シークレットスキャン等、コミットするファイルでは表現できないリポジトリ設定は [`docs/REPO-SETUP.md`](docs/REPO-SETUP.md)(オーナー実行手順)に記載しています。
